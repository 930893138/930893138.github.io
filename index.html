优化原因：性能低，速度慢，sql语句不是最优

注意：sql的编写过程和解析过程不一致

SQL优化：主要是索引的优化

索引：是帮助Mysql 提高校获取数据的数据结构。索引是数据结构（树：B树，hash树）

索引的弊端：

1.索引本身很大，可以存放在内存/硬盘（通常硬盘）

2.索引不是所有情况均适用：a.少量数据频繁修改的字段

c.很少使用的字段

3.索引提升查讯的效率，降低改动的效率

索引的优势：

1，提高查询的效率（降低IO使用率）、

2，降低CPU的使用：

Btree：一般是指B+，数据全部存放在叶节点中

索引分类：

​ 主键索引：不能重复。id 不能是null

​ 单值索引：单例，一个表可以有多个单值索引

​ 唯一索引：不重复。Id， 可以是null

​ 符合索引：多个列构成的索引（相当于二级目录）

创建索引：

方法一：create 索引类型 索引名 on 表（字段）

​ 单值：

​ create index dept_index on tb(dept)

​ 唯一：

​ create unique index name_index on tb（dept）;

​ 符合索引：

​ create index dept_index on tb(dept,name);

方法二：alter table 表名 索引类型 索引名（字段）

​ 单值：

​ alter table td add index dept index(dept)

删除索引：

drop index 索引名 on 表名；

​ 例：drop index name_index on tb ；

查询索引：

​ show index from 表名 ；

​ show index from 表名 \G ；

​ a.分析SQL的执行计划 explain ，可以模拟SQL优化器执行SQL语句，从而让开发人员，知道自己编写的SQL状况

​ b.MySQL查询优化其会干扰我们的优化

查询执行计划： explain+SQL语句

​ explain select * from tb;

事物提交方式：

JDBC：利用JDBC方式处理事物（commit rollback close）【手工】

Mapper动态代理方式的crud（Mybatis接口开发）：

约定：

a.接口，接口中的方法必须遵循以下约定:
1.方法名和mapper. xm1文件中标签的id值相同
2.方法的输入参数和mapper.xml文件中标签的 parameterTypc类型一致3.方法的返回值和mapper.xm1文件中标签的`resultType类型一致
除了以上约定，要实现接口中的方法惇和，Mapper.xml中SQL标签一一对应，还需要以下1点:namespace的值，就是接口的全类名（接口- mapper. xml一一对应)儿配的过程:(约定的过程)
