接口

接口是一个特殊的抽象类，他是一个只有公共抽象方法和全局常量的抽象类。接口是用来解决java中不能多继承的手段，接口在实际使用中更多在实际使用中更多是用来定义功能或者一系列功能的标准。–100%抽象类。

​ 抽象方法：使用abstract修饰的没有方法体的方法，被重写。

​ interface:接口声明

​ implements：接口实现的关键字

接口中的方法默认public abstract修饰。

一个类可以实现多个接口，接口名之间用逗号隔开；

​ 一个类可以继承另一个类。但只能继承一个。

​ 一个接口不被允许继承类。

​ 一个接口允许继承多个接口。

接口中所有的变量都会默认用public static final修饰

接口中不允许存在成员方法，可以存在静态方法（jdk1.8以上才被允许）

1JSP:动态网页

静态、动态：

1.不用 和 是否有“动感”混为一谈
2.是否 随着 时间、地点、用户操作 的 改变而改变

动态网页 需要使用到 服务端脚本语言（JSP）

2架构

CS：Client Server
CS不足：
a.如果 软件升级， 那么全部软件都需要升级
b.维护麻烦：需要维护每一台 客户端软件
c.每一台客户端 都需要安装 客户端软件

BS ：Broswer Server
客户端可以通过 浏览器 直接访问服务端

注意：bs和cs各有优势。

3.tomcat解压后目录：

temp:临时文件
webapps：可执行的项目（将我们开发的项目 放入该目录）
work:存放由jsp翻译成的java,以及编辑成的class文件(jsp ->java ->class)
